# ACO V1 Demo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a runnable Agent Civilization OS V1 demo: World Server + Web (rot.js) + Python Agent Runtime over WebSocket, with map, move/harvest, agent connect, and player select/focus.

**Architecture:** World Server is authoritative (light ECS, tick loop). Frontend observes via WS snapshots and sends `select_agent`. Agent Runtime receives `observation`, returns `action_batch` (mock or LLM). Shared contracts live in `packages/protocol`.

**Tech Stack:** pnpm workspaces, TypeScript (Node 22), `ws`, `better-sqlite3`, Vite + React + rot.js, Python 3.11 + websockets + optional OpenAI-compatible HTTP.

**Spec:** `docs/superpowers/specs/2026-07-15-aco-v1-demo-design.md`

## Global Constraints

- Protocol version string: `"1.0"`
- Map default: 16×12; tick: 2 TPS; agent decision timeout: 2000ms
- Actions only: `move`, `harvest`, `idle`, `say`
- One mutating action per agent per tick; `say` may accompany
- No Redis/Kafka/Postgres; SQLite only for event log
- Agents never mutate world state directly
- Frontend never sends move/harvest in V1
- Full map visibility (no fog)
- Demo ships with 1 agent entity

- [ ] **Step 2: Protocol types (`packages/protocol/src/index.ts`)**

Export exactly:

```ts
export const PROTOCOL_VERSION = "1.0" as const;

export type ActionType = "move" | "harvest" | "idle" | "say";

export interface MovePayload { dx: -1 | 0 | 1; dy: -1 | 0 | 1 }
export interface HarvestPayload {}
export interface IdlePayload {}
export interface SayPayload { text: string }

export type Action =
  | { type: "move"; payload: MovePayload }
  | { type: "harvest"; payload: HarvestPayload }
  | { type: "idle"; payload: IdlePayload }
  | { type: "say"; payload: SayPayload };

export interface ActionBatchMessage {
  type: "action_batch";
  id?: string;
  agentId: string;
  tick: number;
  actions: Action[];
}

export interface WorldEvent {
  type: string;
  payload: Record<string, unknown>;
  tick?: number;
}

export interface ObservationMessage {
  type: "observation";
  protocolVersion: typeof PROTOCOL_VERSION;
  tick: number;
  self: {
    id: string;
    x: number;
    y: number;
    inventory: { ore: number };
  };
  visible: {
    width: number;
    height: number;
    tiles: string[];
    entities: Array<{
      id: string;
      type: "agent" | "resource";
      x: number;
      y: number;
      ore?: number;
      glyph: string;
    }>;
  };
  events: WorldEvent[];
  allowed_actions: ActionType[];
  focused: boolean;
  goal: string | null;
}

export interface SnapshotMessage {
  type: "snapshot";
  tick: number;
  width: number;
  height: number;
  tiles: string[]; // base terrain glyphs only (# .)
  entities: Array<{
    id: string;
    type: "agent" | "resource";
    x: number;
    y: number;
    glyph: string;
    ore?: number;
    inventory?: { ore: number };
  }>;
  focusedAgentId: string | null;
  recentEvents: WorldEvent[];
}

export interface SelectAgentMessage {
  type: "select_agent";
  agentId: string;
}

export interface HelloWorldToAgent {
  type: "hello";
  protocolVersion: typeof PROTOCOL_VERSION;
  agentId: string;
  map: { width: number; height: number };
}

export interface HelloAck {
  type: "hello_ack";
  protocolVersion: typeof PROTOCOL_VERSION;
  runtime: "mock" | "llm";
}

export const ALLOWED_ACTIONS: ActionType[] = ["move", "harvest", "idle", "say"];
```

### Task 3: World tick scheduler + observation + sqlite + WS gateways

**Files:**
- Create: `scheduler/tick.ts`, `observation/buildObservation.ts`, `persist/db.ts`, `gateway/*`, `src/index.ts`, `src/config.ts`

**Interfaces:**
- WS paths (single port 8080):
  - `ws://host:8080/ws/frontend`
  - `ws://host:8080/ws/agent?agentId=agent-1`
- Frontend in: `select_agent` | `ping`
- Frontend out: `snapshot` | `hello` | `error` | `pong`
- Agent in: `hello_ack` | `action_batch` | `ping`
- Agent out: `hello` | `observation` | `error` | `pong`

- [ ] **Step 1: Tick loop**

Every 500ms: apply pending actions → persist events → broadcast snapshot → send observations → clear pending / wait for next action_batch until deadline.

- [ ] **Step 2: SQLite**

Table `events(id INTEGER PK, tick INT, type TEXT, payload TEXT, created_at TEXT)`. Insert each event.

- [ ] **Step 3: Gateways**

Parse JSON; reject unknown; on select set `focusedAgentId`.

- [ ] **Step 4: Integration smoke**

Manual: `pnpm --filter @aco/world dev`, connect with `websocat` or small script, send action_batch, see snapshot.

- [ ] **Step 5: Commit**

```bash
git commit -m "feat(world): tick loop, sqlite log, frontend and agent gateways"
```
